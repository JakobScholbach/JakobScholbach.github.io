    Visualization Guidelines¶

    Interactive visualizations live in wiki/docs/visualizations/ as self-contained HTML files. They are authored here (source-of-truth), flow through the existing MkDocs build, and are deployed alongside the wiki via deploy_wiki_to_personal_site.ps1.

    Color palette¶

    These are the four named colors defined in skript/linear.tex and used throughout the TikZ figures. Use the same colors in visualizations so students see a consistent visual language between the static PDF figures and the interactive web versions.

    Name Hex RGB Role in figures
    myred #bc1e47 188, 30, 71 First / primary vector or object
    myblue #0081cd 0, 129, 205 Second / secondary vector or object
    mygreen #009246 0, 146, 70 Result, sum, or combination
    myyellow #fec247 254, 194, 71 Angles, arcs, auxiliary geometry

    Supporting colors (not in the LaTeX palette but needed for web):

    Role Hex Notes
    Span / region #fec247 at 12% opacity myyellow washed out
    Construction lines #888888 at 55% opacity neutral gray, dashed
    Axes (Three.js default) red / green / blue leave as-is (AxesHelper)

    Semantic rules (derived from TikZ figures in skript/figures/src/): - First vector → myred - Second vector → myblue - Their sum / result → mygreen - Angle or auxiliary arc → myyellow - Parallelogram / construction lines → gray dashed

    Extended palette for visualizations with N > 4 objects (e.g. linear-independence-3d):

    Slot Hex
    5th #7c3aed (purple)
    6th #e06b00 (orange)

    Adding a new visualization¶

    1. Create the HTML file¶

    Place it at wiki/docs/visualizations/<name>.html.
    Name it after the concept, matching the LaTeX label style, e.g. linear-combination-3d.html.

    The file must be self-contained (all JS via CDN importmap for Three.js, or inline for Canvas 2D). Start the <head> with:

    <meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>…</title>

    2. Register it in the LaTeX source¶

    In the relevant .tex subfile, place:

    \visualization{<name>}

    This renders as a small blue clickable link in the PDF (defined in linear.tex) and is replaced by an <iframe> in the wiki by convert_to_wiki.py.

    The \visualization command is a no-op for layout — it does not float or break the paragraph flow. Place it on its own line between paragraphs, typically just before or after the relevant definition or example.

    3. Rebuild the wiki page¶

    python wiki/tools/convert_to_wiki.py --file <subfile-stem>

    The generated wiki/docs/<stem>.md will contain the iframe. No manual edits to .md files.

    Rendering engine: Three.js vs Canvas 2D¶

    There are two rendering approaches in use. Choose based on dimensionality.

    Three.js (all 3D visualizations)¶

    Use the importmap header — pin the version explicitly (@0.160.0). Update all visualizations together when bumping.

    <script type="importmap">
{
  "imports": {
    "three": "https://cdn.jsdelivr.net/npm/three@0.160.0/build/three.module.js",
    "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.160.0/examples/jsm/"
  }
}
</script>
<script type="module">
import * as THREE from 'three';
import { OrbitControls } from 'three/addons/controls/OrbitControls.js';
// …
</script>

    Three.js scene setup (consistent across all 3D files):

    scene.background = new THREE.Color(0xffffff);
scene.add(new THREE.GridHelper(8, 8, 0xe0e0e0, 0xd0d0d0));
scene.add(new THREE.AxesHelper(4));

    Color constants use hex integers:

    const C = {
  red:    0xbc1e47,  // myred   — first vector
  blue:   0x0081cd,  // myblue  — second vector
  green:  0x009246,  // mygreen — result / sum
  yellow: 0xfec247,  // myyellow — angles / auxiliary
  gray:   0x888888,  // construction lines (dashed)
};

    Reference implementation: linear-combination-3d.html.

    Canvas 2D API (all 2D visualizations)¶

    Use a plain <canvas> element and the browser Canvas 2D API — no external library needed.

    <div id="canvas-wrap"><canvas id="canvas"></canvas></div>

    canvas { display: block; touch-action: none; }

    Color constants use CSS strings:

    const C = {
  red:    '#bc1e47',  // myred   — first vector
  blue:   '#0081cd',  // myblue  — second vector
  green:  '#009246',  // mygreen — result / sum
  yellow: '#fec247',  // myyellow — angles / auxiliary
  gray:   '#888888',  // construction lines (dashed)
};

    Reference implementations: eigenvectors-2d.html (standard single-canvas), linear-map-2d.html (dual-pane domain/codomain layout).

    Layout¶

    Standard layout (single canvas)¶

    Full-viewport horizontal split: canvas/3D-view on the left, narrow control panel on the right.

    body { display: flex; height: 100vh; overflow: hidden; }
#canvas-wrap { flex: 1; min-width: 0; }
#panel { width: 270px; flex-shrink: 0; … }

    Panel: width: 270px, background: #f6f8fa, border-left: 1px solid #d0d7de, padding: 16px, gap: 14px–16px.

    Dual-pane layout¶

    For visualizations showing a before/after or domain/codomain split (e.g. linear-map-2d), the canvas area is itself split into two side-by-side panes:

    <div id="canvas-wrap">
  <div class="pane" id="pane-src">…<canvas id="src"></canvas></div>
  <div class="pane" id="pane-tgt">…<canvas id="tgt"></canvas></div>
</div>

    #canvas-wrap { flex: 1; min-width: 0; display: flex; gap: 1px; background: #d0d7de; }
.pane { flex: 1; min-width: 0; position: relative; background: #fff; }

    Use #eef5ff (light blue) for the domain pane background and #f0fff4 (light green) for the codomain pane background.

    Mobile breakpoint¶

    All files include a @media (max-width: 600px) block that stacks the layout vertically:

    @media (max-width: 600px) {
  body { flex-direction: column; height: auto; overflow-y: auto; }
  #canvas-wrap { width: 100%; height: 70vw; min-height: 280px; flex: none; }
  #panel { width: 100%; border-left: none; border-top: 1px solid #d0d7de; overflow-y: visible; }
}

    Canvas height in mobile is typically 55vw–70vw with min-height: 260px–280px.

    Common CSS components¶

    Copy from existing files as needed. All share this base:

    h1 { font-size: 1rem; font-weight: 600; }
.sec { font-size: 0.7rem; text-transform: uppercase; letter-spacing: 0.07em; color: #656d76; margin-bottom: 6px; }
hr { border: none; border-top: 1px solid #d0d7de; }

    Info / result box — monospace output area: 

    .result-box {
  background: #fff; border: 1px solid #d0d7de; border-radius: 6px;
  padding: 10px 12px; font-size: 0.82rem; line-height: 1.8; font-family: monospace;
}

    Preset buttons — small clickable named scenarios: 

    .presets { display: flex; gap: 6px; flex-wrap: wrap; }
button.preset {
  flex: 1 1 auto; padding: 7px 8px; cursor: pointer;
  background: #fff; border: 1px solid #d0d7de; border-radius: 6px;
  font-size: 0.8rem; color: #1f2328; font-family: inherit;
}
button.preset:hover { background: #f0f3f6; border-color: #b0b8c0; }
button.preset:active { background: #e6eaee; }

    Hint text — small instructional copy: 

    .hint { font-size: 0.72rem; color: #656d76; line-height: 1.5; }

    Color swatch — inline legend chip: 

    .swatch { display: inline-block; width: 11px; height: 11px; border-radius: 2px; vertical-align: middle; }

    Matrix input UI — bracket + grid of <input> elements: 

    .mat-outer { display: flex; align-items: center; gap: 1px; }
.bracket { font-size: 2.6rem; color: #888; line-height: 1; }
.mat-inner { display: grid; grid-template-columns: max-content max-content; gap: 6px 8px; padding: 2px 6px; }
.mat-inner input {
  width: 46px; padding: 4px 5px; text-align: center;
  font-family: monospace; font-size: 0.85rem;
  border: 1px solid #d0d7de; border-radius: 5px; background: #fff; color: #1f2328;
}
.mat-inner input:focus { outline: none; border-color: #0081cd; }
